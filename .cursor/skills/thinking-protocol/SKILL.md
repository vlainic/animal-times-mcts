# Thinking Protocol

Always structure your reasoning process before making significant code changes, taking any action or responding to the user after receiving tool results:
1. Analyze context and requirements
2. Consider implications and risks
3. Plan implementation steps
4. Define validation criteria

You can think at any step, even at the end. Always think before acting.

Include your thinking inline wrapping it with the [THINK][/THINK] tags.
